1 Objective

The objective of these instructions is to set up the Centos on your machine as a virtual lab deployment platform.

2 Prerequisites

2.1 Your machine has CentOS version 6.5 or greater running on it.

You can know that by typing 

cat /etc/issue

The output should be 

CentOS release 6.5 (Final)

2.2 Your machine has a 64 bit architecture

To find out, type 

grep flags /proc/cpuinfo

The output should include the word "lm" (for long mode, indicating 64 bit) as one of the flags. 

flags : fpu vme de pse tsc msr pae mce cx8 apic sep mtrr pge mca cmov
pat pse36 clflush dts acpi mmx fxsr sse sse2 ss ht tm pbe nx rdtscp *lm*
constant_tsc arch_perfmon pebs bts xtopology nonstop_tsc aperfmperf
pni dtes64 monitor ds_cpl vmx est tm2 ssse3 cx16 xtpr pdcm pcid sse4_1
sse4_2 popcnt lahf_lm arat dtherm tpr_shadow vnmi flexpriority ept
vpid

2.3 You are on the Internet

The installation pulls packages from open repositories and you will need an Internet connection to run the scripts successfully.

2.4 You have sudo permissions as root

The scripts install software using yum - a package management utility - which can run only in sudo root mode. Therefore, invoke the script with a sudo command. Be careful! Reckless execution of commands with root permission could make your machine inoperable!

3 Instructions for installation

3.1 Changing to the right directory

Please use the structure of the LIK, and change directory to lik-release-0.5.0/ovpl-kit/scripts 

cd lik-release-0.5.0/ovpl-kit/scripts

3.2 Edit the configurations

Edit file config.sh. 

cd scripts
vim config.sh

Edit the config file to set the http_proxy and https_proxy fields. If there are no proxies to be set, leave the fields unchanged. The snippet below shows how to set proxies. 

export http_proxy="http://proxy.vlabs.ac.in:8080"
export https_proxy="http://proxy.vlabs.ac.in:8080"

3.3 Get the openVZ template

cd to the meta directory. Download the openVZ template OS: 

cd ../meta  
wget community.virtual-labs.ac.in/downloads/ubuntu-12.04-custom-x86_64.tar.gz

3.4 Run the script to prepare centos

cd ../scripts
sudo root  ./centos_prepare_ovpl.sh

3.5 Run make to start OVPL

cd ../ovpl/src/
sudo root make

4 Deploy a Lab

At this point, OVPL services are running and you are ready to deploy a lab.

To deploy your lab
  • Open the browser.
  • Enter localhost:8080 as the url.
  • The form requires you to fill Lab ID. You may assign any new alphanumeric name as a lab id.
  • Also, fill in the Repo url with the url of the lab sources on github repository.
  • Click submit to deploy your lab.
  • If the deployment is successful, an IP address will appear in the same browser window.
  • Enter the IP address in the address bar to view the newly deployed lab.
